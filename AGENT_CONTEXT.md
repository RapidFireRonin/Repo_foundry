# Agent Context

Repo Foundry coordinates safe repository creation and maintenance through blueprints, dry-run reconciliation, PR-only changes, local runtime support, dashboard visibility, audit logs, and hourly cycle summaries.

Autonomous-by-default work should continue without routine human approval, but agents must preserve directability through GOALS.md, the direction queue, priorities, policies, and visible logs. PR status snapshots are the standard artifact for connecting pull-request polling, check state, mergeability, policy decisions, dashboard reporting, risks, and rollback notes.
